**精通CSS之为样式找到应用目标**
-----
　　《精通CSS》这本书第二章的题目是：为样式找到应用目标，也就是选择器的应用。利用选择器将CSS样式应用于特定的
HTML元素。  
　　1、最常用的选择器类型是类型选择器和后代选择器。类型选择器用来寻找特定类型的元素，比如段落或标题元素，只需指定希望应用样式的元素的名称，有时候也称为元素选择器或简单选择器。  
```css
p  {color:black;}
h1 {font-weight:blod;}
```
　 2、后代选择器可以用来寻找特定元素或元素组的后代。后代选择器由其他两个选择器之间的空格表示。如
```css
blockquote p {padding-left:2em;}
```  
　　3、利用ID选择器和类选择器可以找到更特定的元素。这两种选择器用于寻找那些具有指定ID或类名的元素。ID选择器由
一个字符#表示，类选择器由一个点号表示。  
　　利用上面提到的四种选择器就可以找到许多元素，如果在文档中添加了很多不必要的类，那么这时需要分析这些元素之间的差异。如果发现唯一的
差异是它们在页面上出现的位置，那么不要给这些元素指定不同的类，而应该将一个类或ID应用于它们的祖先，然后使用后代选择器定位它们。  
　　4、：link和：visited称为链接伪类，只能应用于锚元素。:hover、:active和:focus称为动态伪类，理论上可以应用于任何元素。但是，IE6
只注意应用于锚链接的:active和:hover选择器，完全忽略:focus。IE7在任何元素上都支持:hover,但是忽略：active和：focus。  
　　5、后代选择器选择一个元素的所有后代，而子选择器只选择元素的直接后代，即子元素。IE7和更高版本都支持子选择器。但是，在IE7中有一个小bug,如果
父元素和子元素之间有HTML注释，就会出问题。在IE6和更低的版本中，可以使用通用选择器模拟子选择器的效果。先在所有后代上应用
你希望子元素具有的样式。然后，使用通用选择器覆盖子元素的后代上的样式。我认为可以在需对相同的直接后代添加样式而不影响其他子
元素时利用后代选择器来实现，`. new>li`若存在二级列表则不会受到此样式的影响。  
　　相邻同胞选择器可以用于定位同一个父元素下某个元素之后的元素。比如如果在顶级标题后面的第一个段落的字体要比后续段落稍微大点，可以利用`h2+p` 来选择
元素。同样需要注意的是，与自选择器一样，如果在目标元素之间有注释，在IE7中也会有问题。  
　　6、属性选择器可以根据某个属性是否存在或属性的值来寻找元素。包括IE 7的现代浏览器都支持这些选择器。然而，由于IE 6
不支持属性选择器，可以利用它们对于IE 6应用一种样式，对更符合标准的浏览器应用另一种样式。属性选择器允许根据属性值之一寻找元素。  
　　7、即使在不太复杂的样式中，要寻找同一元素可能有两个或更多规则，CSS通过一个称为层叠的过程处理这种冲突。

* 标有！important的用户样式；  
* 标有！important的作者样式；　
* 作者样式；
* 用户样式；
* 浏览器/用户代理应用的样式。

　　8、可以按逻辑对样式进行分组以及通过添加注释的方法来对代码记性管理。在每个注释头中添加一个标志可以更容易找到目标，如`@group typography`。  
　　9、减小文件的方法可能是启用服务器端压缩。所有现代浏览器都可以处理用GZIP压缩的文件并进行解压。服务器端压缩能够将HTML和CSS
文件减小大约80%，这可以减少对带宽的占用，大大加快页面的下载速度。
